1. Title

Design and Implementation of a Lexical Analyzer in C

2. Objective

To develop a lexical analyzer that reads a C program and identifies tokens such as keywords, identifiers, constants, operators, strings, special symbols, and also reports lexical errors.

3. Introduction

Lexical Analysis is the first phase of a compiler. It scans the source code character by character and groups them into meaningful sequences called tokens.
Examples:

Keywords → int, float, if

Identifiers → num, count

Operators → +, -, ==

Constants → 10, 3.14
It also removes whitespaces, comments, and detects invalid tokens.

4. Scope

This project performs:

Token identification

Lexical error detection

Symbol table generation

Categorization of tokens

Input file reading and scanning

5. System Requirements

Software: GCC/CodeBlocks/VS Code
Hardware: 2 GB RAM, basic processor, 100 MB free disk space

6. Working Methodology

Read the input C source file.

Ignore whitespace, newline, tab, and comments.

Identify tokens using character-by-character scanning.

Categorize tokens into keywords, identifiers, constants, operators, and separators.

Detect invalid characters, invalid identifiers, invalid numbers, and unterminated strings.

Maintain a symbol table for identifiers.

Print token list and errors.

7. Token Categories

Keywords: int, float, char, if, else, while, for, do, return, switch, case, break
Identifiers: Must start with a letter or _
Operators: +, -, *, /, %, =, ==, !=, <=, >=, &&, ||, ++, --
Constants: decimal, hex, octal, float, character
Separators: ( ) { } [ ] ; , . " '

8. Error Handling

The lexical analyzer reports:

Invalid identifiers (2abc, num#1)

Invalid constants (0891, 0x12KZ, 0b1021)

Invalid character constants ('AB')

Unterminated string ("hello)

Illegal symbols (@, ?, <<<)

Missing quotes or unsupported characters

9. System Design (Overview)

Source Code → Lexical Analyzer → Tokens + Errors + Symbol Table
